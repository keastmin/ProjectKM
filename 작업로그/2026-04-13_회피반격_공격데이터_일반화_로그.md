# 2026-04-13 회피반격 공격 데이터 일반화 로그

## 요청 내용
- 회피 반격이 `BasicComboAttackData`를 재사용하고 있었는데, 이를 콤보 공격 전용 구조가 아니라 단일 공격과 스킬 공격에도 사용할 수 있는 방향으로 일반화해달라는 요청.
- 회피 반격에서도 해당 공격 데이터를 실제로 사용해서 공격 이펙트 생성, 적 감지, 데미지 적용까지 동작하게 연결해달라는 요청.

## 변경 이유
- 기존 구조에서는 `BasicComboAttackState` 안에 히트 판정, 이펙트 생성, 데미지 적용 로직이 직접 들어 있어 다른 공격 상태가 재사용하기 어려웠음.
- `BasicComboAttackData`와 `ComboAttackData`라는 이름도 실제 역할보다 좁게 보였고, 회피 반격이나 단타/스킬 공격에도 같은 타이밍 데이터를 쓰려는 요구와 맞지 않았음.
- 회피 반격 상태는 전용 애니메이션만 재생하고 있었고, 타이밍 데이터 기반의 타격 판정과 이펙트 실행은 연결되어 있지 않았음.

## 적용한 구조 변경
- 공통 공격 데이터 베이스 `AttackData`를 추가.
- 공통 공격 타이밍 프로파일 베이스 `AttackTimingProfile`를 추가.
- 공격 타이밍에 따라 히트박스 탐지, 중복 타격 방지, 이펙트 생성, 데미지 적용을 처리하는 공통 실행기 `AttackExecutionRuntime`를 추가.
- 새 일반 목적 공격 데이터 `GeneralAttackData`를 추가해서 이후 단일 공격/스킬 공격도 같은 구조를 사용할 수 있게 확장 포인트를 마련.

## 기존 데이터 호환 방식
- 기존 `BasicComboAttackData`는 삭제하지 않고 `AttackData`를 상속하도록 변경.
- 기존 `ComboAttackData`는 삭제하지 않고 `AttackTimingProfile`을 상속하도록 변경.
- 이 방식으로 기존 에셋 참조를 최대한 유지하면서 런타임 소비 구조만 일반화함.

## 런타임 변경 내용
- `PlayerCore`에서 콤보 데이터 배열과 회피 반격 데이터를 모두 `AttackData` 기준으로 참조하도록 변경.
- `BasicComboAttackState`는 더 이상 자체적으로 타격/이펙트 로직을 들고 있지 않고 `AttackExecutionRuntime`를 사용하도록 정리.
- 콤보 공격은 `ComboAttackData`를 사용할 때만 콤보 입력 가능 시간(`ComboInputStartNormalizedTime`, `ComboInputEndNormalizedTime`)을 읽도록 유지.
- 회피 반격 상태 `DodgeCounterState`는 `_dodgeCounterData`를 읽어:
  - 공격 애니메이션 이름 사용
  - 공격 타이밍 프로파일 사용
  - 공격 이펙트 생성
  - 적 감지 및 데미지 적용
  을 수행하도록 연결.

## 에디터 변경 내용
- `ComboAttackDataEditor`에서 링크 대상 탐색을 `BasicComboAttackData` 전용에서 `AttackData` 기준으로 확장.
- 기존 `BasicComboAttackData`와 새 `GeneralAttackData`를 모두 링크 대상으로 찾을 수 있도록 수정.

## 부가 수정
- `ComboAttackDataEditor`의 스타일 초기화를 `OnEnable()`에서 바로 하지 않고 `OnInspectorGUI()` 시점에 지연 초기화하도록 변경.
- 이유: `EditorStyles.centeredGreyMiniLabel`은 에디터 GUI 스타일이 아직 준비되지 않은 시점에서 접근하면 `NullReferenceException`이 발생할 수 있음.

## 검증
- `dotnet build Assembly-CSharp.csproj` 빌드 통과.
- 빌드 경고 2건은 기존 `AvatarMover` 미사용 필드 경고이며 이번 수정과 무관.
